My experience : a little more about me

My first co-op as a Software Test Engineer at Netgear was not a typical coding job; it was really hands-on, and as a result, I took away a lot more from the experience. There were so many networking concepts I didn’t know going into the job, and I had to figure them out by experimenting and learning as I went. I was assigned tasks like bringing up an entire test chamber on my own, including all the network configurations and topologies. I also had to think about the end goal: what we were trying to achieve and how to make the setup easy and convenient for everyone working in the lab. This experience taught me to be self-reliant, think critically about systems, and develop practical, efficient solutions - skills that have helped me with every other project since.

At the same time, I also ran routine tests on new firmware releases on the devices, and there was a lot of work to do, so as a result a lot of things were delayed. The repetitive tasks were not getting done on time because the more hands-on tasks were higher priority. I noticed that there were ways to make the repetitive testing process more efficient, so I took it upon myself to improve it. The main test that we had to run took about a full work day to run, and I was able to automate the entire process and reduce it down to around 2 hours of offline testing, so you can just start running the test and focus on doing more productive things. Once it was running smoothly, I trained other co-ops so they could continue the workflow. In the end, it was really rewarding to see how much impact small improvements could have on the way the team worked.
